- Опишіть основні принципи роботи з планувальником Cron в ОС Linux. 
 * Як це працює:
У системі постійно працює процес crond, який перевіряє, чи є завдання, що потрібно запустити.
Користувач задає свої завдання у спеціальному файлі — crontab.
Cron порівнює поточний час із розкладом і запускає відповідну команду.
 * Основні команди для роботи:
crontab -e — відкрити список завдань користувача для редагування;
crontab -l — переглянути всі поточні завдання;
crontab -r — видалити всі завдання користувача;
sudo systemctl status cron — перевірити, чи запущено службу cron.

 2.1. Виконання задачі у визначений час (наприклад, о 8:00 ранку)
Мета: автоматичне очищення тимчасової папки /tmp о 8:00 щодня.
Команда: crontab -e
Запис у crontab: 0 8 * * * rm -rf /tmp/*
 Пояснення: 0 8 * * * — кожного дня о 08:00 ранку видаляє всі файли з каталогу /tmp.
 
  2.2. Виконання тієї ж задачі двічі на день
Мета: створення резервної копії каталогу /home/user/Documents о 10:00 і 18:00.
Запис у crontab: 0 10,18 * * * tar -czf /home/user/backup_$(date +\%F_\%H-\%M).tar.gz /home/user/Documents
 Пояснення: Завдання запускається о 10:00 і 18:00. Команда створює архів backup із датою в назві.

 2.3. Виконання задач тільки у будні дні з 8 до 18 години
Мета: запуск перевірки системних логів кожну годину у робочий час.
Запис у crontab: 0 8-18 * * 1-5 cat /var/log/syslog >> /home/user/logs.txt
 Пояснення: 1-5 — понеділок–п’ятниця. 8-18 — з 8:00 до 18:00. Щогодини додає системні логи у файл logs.txt.

3. Використання альтернативного планувальника
 Обрана альтернатива: systemd-timer
Створено сервісний файл: /etc/systemd/system/cleanup.service
 [Unit]
Description=Очистка каталогу /tmp
 [Service]
Type=oneshot
ExecStart=/usr/bin/rm -rf /tmp/*
Створено таймер: /etc/systemd/system/cleanup.timer
 [Unit]
Description=Таймер для очистки каталогу /tmp
 [Timer]
OnCalendar=*-*-* 08:00:00
Persistent=true
 [Install]
WantedBy=timers.target

Активація таймера:
sudo systemctl enable cleanup.timer
sudo systemctl start cleanup.timer

Перевірка статусу: systemctl list-timers
 Пояснення:
Таймер автоматично запускає службу очищення о 8:00 щодня.
Його перевага — логування через journalctl, можливість гнучких умов та більш точний контроль.

 Conclusion
Task scheduling is a crucial part of any operating system, ensuring that repetitive and time-sensitive
operations are executed automatically and reliably. In this work, the Cron scheduler was configured to
perform various automated tasks such as cleaning temporary directories, creating backups, and system
maintenance at predefined times. Additionally, an alternative solution — systemd-timer — was implemented,
providing advanced control, persistent execution, and integrated logging via journalctl.
Through this comparison, it became evident that Cron remains a simple and effective tool for routine automation,
while systemd-timer offers a more modern and flexible approach suitable for complex system management tasks.
