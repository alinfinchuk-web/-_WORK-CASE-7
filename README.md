Що взагалі робить планувальник задач
Планувальник — це така штука в операційній системі, яка дозволяє запускати програми або команди автоматично,
без того щоб ти це робила вручну.

Основні його функції:
* запуск задач у певний час або за певним розкладом;
* повторення задач через певні проміжки часу (щодня, щотижня, щогодини);
* автоматизація рутинних речей — наприклад, очищення тимчасових файлів, резервне копіювання, оновлення системи тощо;
* логування (тобто запис у журнал, що і коли виконалось, щоб потім можна було перевірити).

Порівняння Windows і Linux
У Windows є “Планувальник завдань” (Task Scheduler).
Там усе робиться через графічний інтерфейс — можна просто натиснути “створити задачу”,вибрати коли й що запускати.
Підходить тим, хто не хоче возитися з терміналом.
У Linux для цього використовують cron (старий, перевірений інструмент).
Тут працюєш через термінал — задаєш час і команду в спеціальному файлі (crontab).
Воно гнучке і легке, але треба знати синтаксис.
Отже, Windows зручніша для початківців, бо все видно “віконцями”,
а Linux дає більше контролю і підходить для серверів чи адміністраторів.

 Принцип роботи Cron
Cron — це служба, яка постійно працює у фоні та перевіряє, чи є заплановані задачі. Усі вони записуються у файл crontab.
Щоб відредагувати цей файл, вводять команду: crontab -e

Формат запису такий:
* * * * * команда
│ │ │ │ │
│ │ │ │ └─ день тижня (0–7)
│ │ │ └── місяць (1–12)
│ │ └── день (1–31)
│ └── година (0–23)
└── хвилина (0–59)


 Приклади налаштувань
Виконання задачі о 8 ранку: 0 8 * * * /home/user/scripts/backup.sh
Та сама задача двічі в день (о 8:00 і 18:00): 0 8,18 * * * /home/user/scripts/backup.sh
Тільки у будні з 8 до 18: 0 8-18 * * 1-5 /home/user/scripts/clear_temp.sh
Раз у рік (1 січня): 0 0 1 1 * /home/user/scripts/report.sh
Раз у місяць (1 числа): 0 0 1 * * /home/user/scripts/cleanup.sh
Раз у день: 0 0 * * * /home/user/scripts/update.sh
Щогодини: 0 * * * * /home/user/scripts/autosave.sh
При ввімкненні комп’ютера: @reboot /home/user/scripts/startup.sh

 Альтернатива Cron
Окрім Cron, є інші планувальники:
anacron — виконує задачі навіть якщо комп’ютер був вимкнений у запланований час.
systemd timers — сучасна заміна Cron, має більше можливостей, працює через systemd.
at — використовується для одноразових задач.

Приклад з systemd timer:
створюються два файли: /etc/systemd/system/backup.service
[Unit]
Description=Backup task
[Service]
ExecStart=/home/user/scripts/backup.sh
/etc/systemd/system/backup.timer
[Unit]
Description=Run backup daily at 8AM
[Timer]
OnCalendar=08:00
Persistent=true
[Install]
WantedBy=timers.target

Активується командою:
sudo systemctl enable --now backup.timer
